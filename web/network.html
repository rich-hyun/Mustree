GT 2,000개(당사 1,000·귀사 1,000)로 운영하고, PT는 MST 담보로만 발행(C=10)됩니다.

1) 온보딩

KYC/법인확인 → 라이선스 계약 → 권한 지갑(멀티시그) 등록.

2) 체인 생성

검증자 세팅, 체인 파라미터, GT 2,000개 배분(1,000·1,000).

3) 사용 한도 설정

예치 MST에 따라 월 사용 상한(PT)이 자동 산정됩니다.

필요 예치 계산

공식: 필요 MST = 필요한 PT × C / r (기본 r=500, C=10)

거버넌스(GT)

  • 응급 정지/재개(멀티시그) · 업그레이드 승인
