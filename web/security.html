Security & Encryption Architecture

Vault · AES · HSM 기반 엔터프라이즈 보안 구조 MST-Contract는 클라우드/온프레미스 모두에서 동작하며 1차 AES 암호화 → Vault 키 관리 → HSM 내부 서명의 3단계 보안 체계를 제공합니다.

핵심 보안 구성 요소

Vault (키 금고)

중앙 집중형 키 관리 시스템. 암호화 키 생성·보관·회전을 자동화하며 서버는 절대 키를 보지 않습니다.

AES-256-GCM

가장 안전한 대칭키 방식. 암호화 + 무결성 검사 태그로 데이터 위변조 차단.

HSM (하드웨어 보안 모듈)

서명 전용 장치. 개인키 외부 반출 금지. 트랜잭션 서명은 항상 HSM 내부에서 수행.

보안 도입 로드맵

1단계

전송 계층 보호 + 요청 서명
TLS 1.3 / HMAC / Idempotency-Key

2단계

AES 암호문 저장/조회
Vault Transit / AES-GCM

3단계

개인키 외부 반출 금지
HSM 서명 / PKCS#11

4단계

Webhook/로그 무결성
HMAC 서명, 해시체인

Webhook 결과 알림

  • 지수 백오프 자동 재시도
  • 멱등키 기반 중복 방지
  • HMAC-SHA256 서명 검증
X-Signature: HMAC-SHA256=58af0c3e...

감사 로그 & 추적

모든 요청·서명·결과는 해시체인 기반으로 기록되어 변경 불가하며 대시보드에서 조회 가능합니다.

RBAC 역할 기반 권한

운영자/개발자/검증자 등 역할별로 민감 API 접근을 제한합니다.

온프레미스 및 데이터 주권 옵션

기업 보안정책에 따라 Vault 및 HSM을 내부망에 배치할 수 있습니다. 동일한 API 프로토콜을 유지하면서 데이터 주권을 완전히 보존할 수 있습니다.
