Security & Encryption Architecture

Vault · AES · HSM 기반
엔터프라이즈 보안 구조

MST-Contract는 클라우드와 온프레미스 모두에서 작동 가능한
1차 AES 암호화 + Vault 키 관리 + HSM 내부 서명 체계를 제공합니다.

보안 아키텍처 개요

데이터 암호화, 키 관리, 서명, 결과 전달 등 각 단계별로 보안이 내장되어 있습니다.

보안 & 암호화 설계

구간 목적 방식(1차) 구현 예시 비고
클라이언트 ↔ API 전송 보호 TLS 1.3 + 요청 서명 Nginx/Cloudflare + HMAC/JWT 멱등키 (Idempotency-Key) 중복 방지
API ↔ Vault 암호화 서비스 호출 Vault Transit API POST /v1/transit/encrypt 서버는 키 평문 비보관
API ↔ DB/IPFS 민감데이터 보호 AES-256-GCM (암호문 저장) Vault가 암복호화 수행 IPFS는 암호문만 저장
API ↔ HSM 서명 키 보호 HSM 내부 서명 PKCS#11 / CloudHSM 키 외부 반출 금지
Webhook 결과 위조 방지 HMAC-SHA256 서명 X-Signature 헤더 재전송 백오프 + 멱등성
로그/백업 유출 대응 해시체인 + AES WORM/S3-SSE 감사 추적 무결성

Vault (키 금고)

암호화 키 생성·보관·회전을 자동화하는 중앙 보안 시스템입니다. 모든 데이터는 Vault API를 통해 암·복호화 되며 서버에는 키가 남지 않습니다.

AES-256-GCM

가장 안전한 대칭키 암호화 방식입니다. Vault 내부 및 애플리케이션 단에서 데이터를 암호화하며, 무결성 검증 태그로 위변조를 차단합니다.

HSM (하드웨어 보안 모듈)

서명 및 키 운영을 물리적으로 분리합니다. 블록체인 서명, 거래 승인 등은 항상 HSM 내부에서 수행되어 키 유출 위험이 없습니다.

도입 로드맵

단계적용 대상핵심 목표구현 포인트
1단계API → 서버전송 계층 보호 + 요청 서명TLS 1.3, HMAC/JWT, Idempotency-Key
2단계서버 ↔ DB/IPFSAES 암호문 저장/조회Vault Transit, AES-GCM, 키 비노출
3단계HSM 연동개인키 외부 반출 금지Signer 분리, PKCS#11, 승인정책
4단계Webhook/로그무결성·전달 보장서명 검증, 백오프, 해시체인

결과 알림 (Webhook)

전달 흐름

  • 멱등키 기반 중복 방지
  • 지수 백오프 자동 재시도
  • 서명 검증(HMAC) 무결성 보장
  • 대시보드 엔드포인트 관리 지원

서명 헤더 예시

X-Signature: HMAC-SHA256=58af0c3e…

수신 서버는 동일 Secret으로 해시를 계산하여 데이터 위변조를 검증합니다.

감사 로그 및 추적

모든 서명·요청·결과는 해시체인 로그로 기록되어 변경 불가합니다. 관리자는 대시보드에서 로그 조회 및 내보내기가 가능합니다.

역할 기반 권한 관리 (RBAC)

운영자, 개발자, 검증자 등 역할별 권한이 분리되어 민감 API 및 Vault 접근을 통제합니다.

온프레미스 및 데이터 주권 옵션

기업 보안정책에 따라 Vault 및 HSM을 내부망에 설치할 수 있습니다. 클라우드 서비스와 동일한 API를 사용하며 데이터 주권을 유지합니다.
